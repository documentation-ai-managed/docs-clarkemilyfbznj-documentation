## Overview

You can build comprehensive docs sites with Markdown, MDX, and powerful components while maintaining version control and collaboration features. This space helps teams document APIs, user guides, and internal knowledge bases efficiently.

## Brand Guidelines

Maintain consistency across your documentation using Clarkemilyfbznj's primary brand color `#3B82F6`. Apply it to headings, links, and accents for a professional look.

<Expandable title="Advanced Brand Customization" default-open="false">

````css

  --brand-primary: #3B82F6;
  --brand-text: #1E293B;
````

Apply in your theme configuration file for buttons and highlights.

## Initial Setup

Get your documentation site running quickly. Follow these steps to initialize a new project.

<Steps>
  <Step title="Create Project" icon="plus">
    Use the CLI to bootstrap your site.

    <CodeGroup tabs="npm,yarn">
    ````bash
    npm create clarkemilyfbznj@latest my-docs-site
    ````

    ````bash
    yarn create clarkemilyfbznj my-docs-site
    ````
    </CodeGroup>
  </Step>

  <Step title="Install Dependencies" icon="download">
    Navigate to your project and install packages.

    <Tabs>
      <Tab title="Node.js" icon="nodejs">
      ````bash
      cd my-docs-site
      npm install
      ````
      </Tab>
      <Tab title="pnpm" icon="package">
      ````bash
      cd my-docs-site
      pnpm install
      ````
      </Tab>
    </Tabs>
  </Step>

  <Step title="Start Development Server" icon="play">
    Launch the local server to preview changes.

    ````bash
    npm run dev
    ````

    Open `http://localhost:3000` in your browser.
  </Step>
</Steps>

<Callout kind="tip">
  Customize your sidebar and theme in `config.json` after setup. Explore `/docs/configuration` for details.
</Callout>
